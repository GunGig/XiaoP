✍️任务1：小P的第一次“书写”尝试

想象一下，我们要让小P在一张白纸 (一个新的文本文件) 上写下第一句话。比如：“今天天气真好！”

核心魔法指令：

  • open("文件名", "w"): 打开一个文件，"w" 表示“写入模式 (write)”。如果文件不存在，它会创建这个文件；如果文件已存在，它会清空里面的内容，准备重新写入！
  • 文件对象.write("你想写的内容"): 把文字写进打开的文件里。
  • 文件对象.close(): 写完后一定要关闭文件，就像写完作业要合上本子一样，不然内容可能保存不成功哦！

➕任务2：让小P学会“追加”记录

每次都覆盖掉以前的内容可不行，我们希望小P能把新的笔记加在旧笔记的后面，而不是擦掉重写。

新的魔法指令：

  • open("文件名", "a"): 这里的"a"表示“追加模式 (append)”。如果文件不存在，它会创建文件；如果文件存在，它会在文件末尾添加新内容，不会动原来的东西！
  • 别忘了换行符 \n！每次写完一条笔记，最好在末尾加上\n，这样下一条笔记就会从新的一行开始，看起来整齐。

🛡️任务3：更安全的“保险箱”写法 (`with`语句)

每次都写file.close()有点麻烦，万一忘了呢？Python提供了一个更棒的方法：with语句。它像一个保险箱，能自动帮我们关好文件，即使程序出错也不会忘记！

`with`语句的魔法： with open("文件名", "模式") as 文件别名:

  • 在with代码块里做的文件操作，结束后Python会自动帮你close()文件。
🧑‍💻你的回合：

把任务2的代码用with语句改写一下，实现同样的功能：向my_notes.txt追加一条新的待办事项，比如“明天要记得复习Python！”。

🤔动动小脑筋：

使用with语句后，我们还需要手动写file.close()吗？为什么说它更安全？

✅参考答案与解析：

不需要手动写file.close()了！with语句会在代码块结束时（无论正常结束还是发生错误）自动关闭文件，所以更安全，也能防止因为忘记关闭文件导致的数据问题。

# 任务3的参考代码
new_task = "明天要记得复习Python！\n"

with open("my_notes.txt", "a", encoding="utf-8") as my_file_alias:
    my_file_alias.write(new_task)

print(f"小P用'with'语句安全地记下了：{new_task.strip()}") # strip()去掉末尾换行符方便打印

函数的好处：

✅参考答案与解析：

# 任务4的参考代码

def add_memo(memo_text):
    """这个函数会把一条备忘录追加到文件中。"""

    print(f"小P已添加备忘：'{memo_text}'")

🌍科普小贴士：文件系统和文本文件
  • 文件系统 (File System): 就像我们书包里的不同科目文件夹，电脑用文件系统来管理各种文件和文件夹 (也叫目录)。我们创建的my_notes.txt就是文件系统中的一个文件。
  • 文件 (File) & 文件夹 (Folder/Directory): 文件是存储信息的基本单位，文件夹是用来组织文件的容器。
  • 路径 (Path): 告诉电脑去哪里找到特定文件或文件夹的“地址”，比如C:\Users\你的名字\Documents\my_notes.txt。
  • 文本文件 (.txt): 是一种非常简单的文件，里面存储的就是我们能直接看懂的文字和符号（比如字母、数字、标点）。小P的记事本就是这种文件。
  • `encoding="utf-8"`: 这是一个小魔法，告诉电脑用一种通用的“密码本”（编码方式）来读取和写入文字，这样我们写的中文才不会变成奇怪的乱码。

🌟第二课时：让小P学会“读”日记 (读取文件与整合)

📖任务5：小P“朗读”所有记录

我们希望小P能一次性把my_notes.txt里的所有笔记都读出来，显示在屏幕上。

核心魔法指令：

  • open("文件名", "r"): "r" 表示“读取模式 (read)”。
  • 文件对象.read(): 一口气读取文件里所有内容，返回一个包含所有文字的大字符串。
  • 文件对象.readlines(): 读取文件里所有行，返回一个列表 (list)，列表中的每个元素是文件里的一行文字（通常带着末尾的换行符\n）。
🧑‍💻你的回合：

试着编写代码，用"r"模式打开my_notes.txt。
1. 先用file.read()尝试读取并打印。
2. 再用file.readlines()尝试读取，并用一个for循环把列表里的每一条笔记打印出来。观察它们的区别。

🤔动动小脑筋：

read() 和 readlines() 都能读取内容，它们有什么不同？如果我们想一条一条地处理笔记（比如给每条笔记编号），哪种方法可能更方便一些？打印readlines()的结果时，可能会发现每行末尾多了个空行，这是为什么呢？（提示：想想写入时加了什么）

✅参考答案与解析：

read() 返回一个包含所有内容的单一字符串。readlines() 返回一个字符串列表，每行是列表的一个元素。对于逐条处理，readlines() 更方便。多余的空行是因为每行文字本身末尾有\n，而print()函数默认也会在打印后换行，所以就多了一行。我们可以用字符串的.strip()方法去掉多余的空白和换行符。

# 任务5的参考代码
MEMO_FILENAME = "my_notes.txt"

print("--- 方法1: 使用 read() ---")
try: # 使用 try-except 避免文件不存在时报错
    with open(MEMO_FILENAME, "r", encoding="utf-8") as file:
        all_content = file.read()
        if all_content:
            print(all_content)
        else:
            print("记事本是空的哦！")
except FileNotFoundError:
    print(f"哎呀，找不到 {MEMO_FILENAME} 文件！小P还没开始写日记呢。")

print("\\n--- 方法2: 使用 readlines() ---")
try:
    with open(MEMO_FILENAME, "r", encoding="utf-8") as file:
        memo_lines = file.readlines() # 这是一个列表
        if memo_lines: # 判断列表是否为空
            print("小P的备忘录内容：")
            for line in memo_lines:
                print(line.strip()) # strip() 去掉首尾空白，包括换行符
        else:
            print("记事本是空的哦！")
except FileNotFoundError:
    print(f"哎呀，找不到 {MEMO_FILENAME} 文件！")

如果在调用view_memos()之前，我们一次都还没添加过备忘 (即my_notes.txt文件还不存在)，我们的函数应该怎么处理这种情况才不会报错呢？

✅参考答案与解析：

应该使用try-except FileNotFoundError来捕获文件不存在的异常，并给用户一个友好的提示，而不是让程序崩溃。

# 任务6的参考代码
MEMO_FILENAME = "my_notes.txt"

def view_memos():
    """这个函数会读取并展示所有备忘录。"""
    try:
        with open(MEMO_FILENAME, "r", encoding="utf-8") as file:
            memos = file.readlines()

        if not memos: # 检查列表是否为空
            print("小P的记事本还是空的，快去添加一些吧！")
            return # 如果为空，直接结束函数

        print("\\n--- 小P的超级记事本 ---")
        for index, memo in enumerate(memos):
            print(f"{index + 1}. {memo.strip()}") # index从0开始，所以+1
        print("-----------------------")

    except FileNotFoundError:
        print(f"小P的记事本 ({MEMO_FILENAME}) 还没创建呢！先添加一条备忘吧。")

# 任务7的参考代码
# (假设 add_memo 和 view_memos 函数已经定义好了，就像上面任务4和任务6那样)
MEMO_FILENAME = "my_notes.txt"

def add_memo(memo_text):
    with open(MEMO_FILENAME, "a", encoding="utf-8") as file:
        file.write(memo_text + "\\n")
    print(f"小P已添加备忘：'{memo_text}'")

def view_memos():
    try:
        with open(MEMO_FILENAME, "r", encoding="utf-8") as file:
            memos = file.readlines()
        if not memos:
            print("小P的记事本还是空的！")
            return
        print("\\n--- 小P的超级记事本 ---")
        for index, memo in enumerate(memos):
            print(f"{index + 1}. {memo.strip()}")
        print("-----------------------")
    except FileNotFoundError:
        print(f"记事本 ({MEMO_FILENAME}) 还没创建呢！")

# --- 主程序开始 ---
print("欢迎使用小P的超级记事本！")
while True:
    print("\\n你可以做什么：")
    print("1. 添加新的备忘")
    print("2. 查看所有备忘")
    print("3. 退出记事本")

    choice = input("请输入你的选择 (1/2/3): ")

    if choice == '1':
        memo = input("请输入你要记录的内容: ")
        if memo: # 确保用户输入了内容
            add_memo(memo)
        else:
            print("你没有输入任何内容哦。")
    elif choice == '2':
        view_memos()
    elif choice == '3':
        print("感谢使用小P记事本，下次再见！👋")
        break # 退出循环
    else:
        print("无效的选择，请输入数字 1, 2 或 3 哦！")

💡科普再回顾：文本文件如何存储信息？

我们的小P记事本 (my_notes.txt) 是一个文本文件。它里面存储的就是我们输入的字符序列。当你用write()写入时，这些字符就被按照一定的编码（我们用了UTF-8）转换成计算机能理解的二进制数据存到硬盘上。当你用read()或readlines()读取时，这些二进制数据又被解码回我们能看懂的字符。每一行用换行符\n分隔，就像我们写信时换行一样，让内容清晰可读。
